# 个人量化套利中枢系统设计文档

> 日期：2026-04-29
> 版本：V1（四策略集成：可转债打新 + 可转债配债 + 节假日逆回购 + LOF底仓套利）
> 审计修正：2026-04-29（基于方案探索3审计结论修正）

---

## 1. 项目概述

### 1.1 目标

构建一个个人多策略量化中枢，集成四个互不冲突的低风险套利策略，让资金永远不闲着、精力接近零消耗：

1. **可转债打新**：每日一键申购，零资金占用，中签后缴款卖出
2. **可转债配债（抢权配售）**：监控即将发行转债，筛选安全垫达标的品种，股权登记日前买入正股锁定配债额度
3. **节假日逆回购套利**：节前自动做逆回购，赚取假期利息
4. **LOF底仓套利**：实时监控溢价率，触发时卖出底仓+申购，瞬间锁定价差

### 1.2 核心设计哲学

**高收益无风险套利机会一旦被发现，很快就会因资金涌入而消失。** 系统的核心价值不在于找到"最好的策略"，而在于建立一套能快速发现、快速执行的系统——有系统的人先吃到，没系统的人只能看限购公告。

### 1.3 策略互补逻辑

| 维度 | 可转债打新 | 可转债配债 | 节假日逆回购 | LOF底仓套利 |
|------|-----------|-----------|-------------|------------|
| 资金占用 | 0（信用申购） | 短（1-2天持正股） | 短（1-7天） | 长（底仓长期持仓） |
| 时间消耗 | 极低（3秒/天） | 低（每月1-2次，每次几分钟） | 极低（节前5秒） | 中（等信号，偶尔确认） |
| 风险敞口 | 极低（首日抛，破发极少） | 低（安全垫可量化，如6.45%） | 几乎为零 | 低（T+0锁定价差） |
| 操作频率 | 每日 | 每月1-2次 | 每年7-8次 | 偶尔脉冲 |
| 收益逻辑 | 散户盲打红利 | 持仓锁定配额，确定性远高于打新 | 流动性溢价 | 瞬时溢价+技术监控差 |

四者亏损因素几乎正交，同时亏损概率趋近于零。打新是"凭运气中签"，配债是"凭持仓锁定"，两者天然互补。

### 1.4 资金分配（以10万为例）

| 用途 | 金额 | 说明 |
|------|------|------|
| LOF底仓 | 5-8万 | 以宽基LOF为主（沪深300/中证500），辅以1-2只溢价频发的行业LOF，定投方式建仓 |
| 日常可用 | 2-5万 | 中签缴款、配债买入正股、临时加仓 |
| 可转债打新 | 0 | 信用申购，中签后从可用资金划出约1000元 |
| 配债正股 | 临时调配 | 股权登记日前1天买入正股，T+1日卖出，资金1-2天回归 |
| 逆回购 | 节前临时调配 | 节前用闲余资金做逆回购，**至少预留20%可变现资金**覆盖突发需求，节后自动回归 |

### 1.5 版本规划

| 版本 | 范围 | 状态 |
|------|------|------|
| V1.1 | 基础框架 + 交易日历 + 可转债打新脚本 | 本次首批实现 |
| V1.2 | 逆回购策略 + 节假日检测 | 本次第二批实现 |
| V1.2.5 | 开通miniQMT并测试连通性 | 期间并行 |
| V1.3 | LOF溢价率监控 + **自动交易**（直接升级，不再手动抓信号） | 本次第三批实现 |
| V1.4 | 可转债配债策略（含保守估值、抢权避让） | 本次第四批实现 |
| V1.5 | 四策略统一看板 + 全天候综合调度 | 本次集成 |
| V2 | 四策略全自动化 + V2合规报备 | 后续迭代 |

实现顺序：可转债打新（验证环境）→ 逆回购（验证定时任务）→ **开通miniQMT** → LOF监控+自动交易（溢价窗口几分钟，手动抓不住）→ 配债（验证数据筛选+事件驱动）→ 统一看板

> **关键变更**：LOF套利窗口通常只有几分钟，人工收到通知再翻App操作抓住的概率不到30%，因此将miniQMT自动交易提前到V1.3。

## 3. 策略调度器

### 3.1 职责

- 管理各策略的生命周期（启动、暂停、停止）
- 根据交易日历判断是否为交易日
- 定时触发各策略的执行逻辑
- 事件驱动触发（如配债的股权登记日提醒）
- 统一管理策略状态和运行日志

### 3.2 调度规则

| 策略 | 触发条件 | 触发时间 | 执行方式 |
|------|----------|----------|----------|
| 可转债打新 | 每个交易日 | 9:30 | 获取当日可申购列表，自动提交申购 |
| 可转债配债 | 有新转债发行公告时 | 每日检查 | 筛选配债机会 → 登记日前1天提醒买入正股 |
| 节假日逆回购 | 长假前最后一个交易日 | 14:30附近（利率通常最高） | 提醒+一键下单逆回购 |
| LOF底仓套利 | 每个交易日的交易时段 | 9:30-15:00 持续 | 5-8秒轮询溢价率+随机延迟，触发自动交易 |

### 3.3 交易日历

- 维护A股交易日历（排除周末、法定节假日）
- 提前标记长假节点（春节、国庆、五一等）
- 标记季末/年末时点（逆回购利率可能冲高）
- 维护转债发行日历（股权登记日、申购日、上市日）

## 4. 数据层

### 4.1 数据源

| 数据 | 来源 | 频率 |
|------|------|------|
| 交易日历 | 上交所官方日历 / AkShare / 内置日历 | 启动时加载，每年校准 |
| LOF基金列表 | AkShare 基金名录接口 | 每日1次 |
| LOF实时行情 | 主源AkShare / 备用新浪财经、腾讯行情 / xtquant | 5-8秒轮询+0-3秒随机延迟 |
| LOF IOPV | 主源：东方财富/集思录公开实时IOPV接口 / 备用：xtquant | 5-8秒轮询+0-3秒随机延迟 |
| LOF基金净值 | AkShare | 每日1次 |
| 可转债申购列表 | AkShare 可转债接口 | 每日9:25 |
| 中签结果 | AkShare / 手动查询 | T+2日 |
| 可转债发行计划 | AkShare / 集思录 | 每日1次 |
| 正股行情（配债用） | AkShare | 配债筛选时实时获取 |
| 正股基本面（配债排雷） | AkShare | 配债筛选时获取 |
| 逆回购利率行情 | AkShare | 节前14:00起实时监控 |

> **IOPV获取策略（关键修正）**：原方案"前日净值+成分股涨跌幅估算"误差0.5%-2%，会淹没2%阈值产生大量虚假信号。现改为优先使用东方财富/集思录公开实时IOPV接口，精度远高于自估算。若接口不可用则回退到xtquant获取券商IOPV。仅当所有实时源均不可用时才使用自估算，此时阈值自动提升到3%并标注"低精度模式"。
>
> **数据源容灾**：每个关键接口配一个备用源（主源AkShare，备用新浪财经/腾讯行情），连续失败3次立即切换并告警通知。

### 4.2 数据存储（SQLite）

核心表：

- **lof_fund**：基金代码、名称、状态、是否限购、日成交额
- **premium_history**：时间戳、基金代码、场内价格、IOPV、溢价率
- **trade_signal**：触发时间、基金代码、溢价率、操作建议、状态（已通知/已执行）
- **position**：底仓基金代码、份额、成本价（V2用）
- **bond_ipo**：转债代码、名称、申购日、中签结果、缴款状态、上市日、卖出状态
- **bond_allocation**：转债代码、正股代码、正股名称、百元含权、安全垫、股权登记日、配债缴款日、上市日、状态（待买入/已买入正股/已卖出正股/已配债/已上市卖出）
- **reverse_repo**：日期、品种代码、利率、金额、到期日、收益
- **holiday_calendar**：日期、是否交易日、是否长假前最后交易日、节假日名称
- **daily_summary**：日期、策略类型、当日盈亏、操作记录
- **data_source_status**：数据源名称、状态（正常/故障）、最后成功时间、连续失败次数

### 4.3 采集策略

- **LOF行情**：交易时间（9:30-15:00）以5-8秒+0-3秒随机延迟间隔轮询全市场，防反爬
- **数据源容灾**：主源连续失败3次立即切换备用源，同时推送告警通知
- **可转债数据**：每日9:25获取当日可申购列表
- **配债数据**：每日1次获取转债发行计划，筛选时实时获取正股行情
- **逆回购利率**：节前14:00起实时监控，14:30附近提醒操作（利率通常最高）
- **日初同步**：每个交易日开盘前同步基金列表、昨日净值、交易日历、转债发行日历
- **数据保留**：溢价率历史仅保留近3个月明细，更早数据按小时聚合，定期VACUUM数据库

## 5. 策略1：可转债打新

### 5.1 核心流程

```
每日9:25 → 获取当日可申购转债列表
         → 9:30 自动提交申购（沪市1000手/深市1000张，顶格申购）
         → 微信/桌面通知申购结果
         → T+2日自动查询中签结果
         → 中签 → 通知缴款（约1000元/签）
         → 上市首日 → 通知卖出
```

### 5.2 规则与风控

- **顶格申购**：沪市上限100万（1000手），深市上限10万（1000张）
- **违约约束**：连续12个月违约3次将暂停打新资格，中签必须缴款
- **违约保护**：T+2查询到中签时，立即核对可用资金；资金不足时优先缴款沪市（收益更高）；违约累计达2次时自动暂停打新，防止被暂停资格
- **上市卖出**：上市首日开盘卖出，不持过夜
- **破发处理**：若上市首日破发，仍建议卖出（亏损有限，避免持有风险）
- **账户区分**：沪市和深市的申购上限和代码规则不同，需分别处理

### 5.3 预期收益

- 年均中签5-10签，每签赚100-300元
- 年收益约500-3000元，风险调整后收益极高

## 6. 策略2：可转债配债（抢权配售）

### 6.1 核心逻辑

配债是指上市公司发行可转债时，现有股东在股权登记日持有正股，即可按持股比例优先获得转债配额，无需参与网上抽签，相当于"定向中签"。

与打新的区别：打新是"凭运气中签"，配债是"凭持仓锁定"——确定性远高于打新。

### 6.2 核心流程

```
每日检查 → 获取即将发行转债列表
         → 自动筛选（百元含权>20 且 安全垫>5% 且 正股无ST/退市风险）
         → 达标品种入库 → 股权登记日前1天推送提醒
         → 提醒买入正股 → T+1日卖出正股 + 缴款配债
         → 转债上市首日 → 通知卖出
```

### 6.3 筛选条件

| 指标 | 阈值 | 说明 |
|------|------|------|
| 百元股票含权 | > 20元 | 每100元市值能配到的转债面值，越高配债效率越高 |
| 安全垫 | > 5% | 配债收益能覆盖正股下跌的最大比例，如6.45%意味着正股跌6.45%以内仍不亏 |
| 正股质地 | 非ST、无退市风险 | 自动排雷，获取正股基本面数据做快速过滤 |
| 正股流动性 | 日成交额 > 1000万 | 确保正股能顺利买入卖出 |

**安全垫计算（保守估值）**：
```
预估上市价 = 面值100 × (1 + 近3月同评级同转股价值转债平均开盘溢价率 × 0.8保守系数)
预估收益 = (预估上市价 - 100) × 可配张数
安全垫 = (预估收益 / 买入正股金额) × 100%
```
内置0.8保守系数，避免自我欺骗。系统自动拉取近3月同评级、同转股价值转债的开盘溢价率作为基准。

### 6.4 操作时间线

```
D-1日（股权登记日前一天）→ 提醒买入正股
D日（股权登记日）        → 持有正股收盘，获得配债权 → 尾盘卖出正股（避开次日集中抛售）
D+1日                   → 缴款配债（正股已在D日尾盘卖出，资金已回笼）
D+N日（转债上市日）      → 卖出转债
```

正股资金占用仅1-2天，资金快速回归。

### 6.5 规则与风控

- **安全垫底线**：只参与安全垫>5%的品种（可配置），不追低安全垫机会
- **保守估值**：预估上市价使用近3月同评级同转股价值转债平均开盘溢价率×0.8保守系数计算
- **正股排雷**：自动排除ST、*ST、退市风险股
- **抢权避让**：股权登记日前若正股已涨超5%，系统标记"高滑点预警"，谨慎参与
- **尾盘卖出**：正股在登记日尾盘卖出，避开D+1日集中抛售导致的滑点
- **单笔上限**：单只配债正股买入金额不超过总资金的20%
- **上市卖出**：转债上市首日开盘卖出，不持过夜（与打新一致）
- **滑点记录**：系统记录每笔配债正股的实际滑点（买入价vs卖出价），高滑点品种历史数据用于优化筛选

### 6.6 预期收益

- 每月1-2次配债机会，单签收益约300-700元
- 2025年近六成新债上市收盘价超140元，优质品种首日涨至157.3元上限
- 年收益约2000-8000元，确定性远高于打新

## 7. 策略3：节假日逆回购套利

### 7.1 核心流程

```
节前最后交易日 → 检测到长假节点
              → 14:00起实时监控逆回购利率
              → 14:30附近提醒操作（利率通常最高）
              → 计算最佳品种（1天期 vs 多天期）
              → 检查资金日历（未来7天资金需求），至少预留20%可变现资金
              → 一键下单
              → 节后资金自动到账
```

### 7.2 品种选择

| 品种 | 代码 | 门槛 | 说明 |
|------|------|------|------|
| 沪市1天逆回购 | 204001 | 10万起 | 节前利率通常最高 |
| 深市1天逆回购 | 131810 | 1000起 | 门槛低，适合小额资金 |
| 沪市7天逆回购 | 204007 | 10万起 | 长假可选 |
| 深市7天逆回购 | 131801 | 1000起 | 长假可选 |

**选择逻辑**：节前最后交易日做1天期逆回购最划算（实际计息天数包含整个假期）。

### 7.3 触发时机

- 长假前最后交易日（春节、国庆、五一等）
- 季末/年末时点（逆回购利率可能冲高，即使非长假也值得关注）
- 14:00起实时监控利率，14:30附近提醒操作（利率通常最高）
- 资金预留：至少预留20%可变现资金，覆盖突发的打新中签缴款或配债需求
- 资金日历：自动拉取未来7天资金需求（中签缴款日、配债缴款日等），避免资金锁死

### 7.4 预期收益

- 每次长假年化利率可达5-15%（节前利率脉冲）
- 一年约7-8次长假，10万本金年增收约500-800元

## 8. 策略4：LOF底仓套利

### 8.1 溢价率计算

```
溢价率 = (场内价格 - IOPV) / IOPV × 100%
```

- 场内价格：实时买入一档价
- IOPV：实时估算净值

### 8.2 阈值策略

- **默认阈值**：3%（基于实时IOPV接口时可用2%，低精度模式下自动提升到3%）
- **低精度模式**：当实时IOPV源不可用、回退到自估算时，阈值自动提升到3%，通知中标注"低精度模式，建议人工确认"
- **动态阈值**：可按基金单独配置（流动性差的基金阈值需更高）
- **过滤条件**：
  - 日成交额 > 500万（排除流动性不足的基金）
  - 申购未暂停（排除限购基金）
  - 单日限购额度足够（限额不够时不触发）
  - 底仓持有该基金（自动交易必须满足）

### 8.3 信号生成

当溢价率超过阈值时，生成交易信号包含：

- 基金代码/名称
- 当前溢价率
- 场内价格 / IOPV
- 建议操作（卖出底仓 + 申购同等份额）
- 预估利润率（扣除费用后）

### 8.4 防抖机制

- 同一只基金5分钟内不重复发送信号
- 溢价率必须连续3个采样点超过阈值才触发（过滤瞬时噪声）

### 8.5 底仓风控

- **底仓选择**：优先选沪深300/中证500等宽基LOF（市场风险分散），辅以1-2只溢价频发的行业LOF
- **定投建仓**：底仓用定投方式逐步建仓，避免一次性重仓
- **回撤止损**：底仓设15%回撤硬止损，触发时减仓或清仓，保护本金
- **盈亏展示**：看板同时展示套利收益和底仓浮动盈亏，避免账目失真

### 8.6 LOF套利 vs 逆回购冲突仲裁

若当天既是节前最后交易日、又有LOF溢价信号，系统自动比较两者预期收益：
- LOF溢价>3%时，优先保住LOF脉冲机会（收益远高于逆回购）
- LOF溢价1-3%时，优先做逆回购（确定性收益更高）
- 冲突时推送对比信息，由用户最终确认

## 10. 交易层

### 10.1 V1.3起：LOF自动交易

V1.3阶段开通miniQMT后，LOF套利直接使用自动交易（手动操作抓住溢价窗口的概率不到30%）：

- **自动卖出底仓**：溢价信号触发后自动市价卖出底仓份额
- **自动申购同等份额**：场内申购同等金额
- **执行顺序保护**：**必须先收到卖出成交回报，再发起申购**，杜绝裸卖空风险
- **执行确认**：检查成交回报，确认卖出和申购都成功
- **异常处理**：卖出成功但申购失败时，记录告警并通知人工介入

### 10.2 V2：全策略自动化

V2阶段将所有策略升级为自动交易：

- **可转债打新**：自动申购 + 中签缴款 + 上市首日自动卖出
- **可转债配债**：股权登记日前自动买入正股 + 登记日尾盘自动卖出正股 + 自动缴款配债 + 上市首日自动卖出
- **逆回购**：节前自动下单逆回购

### 10.3 V2合规报备

V2上线自动交易前，主动向券商报备程序化交易，避免合规风险。

## 16. 开通miniQMT步骤（V1.2.5前置，V1.3必需）

> **重要**：LOF套利窗口通常只有几分钟，手动操作抓住概率不到30%，因此miniQMT必须在V1.3之前开通。

1. 选择华鑫证券或国金证券开户
2. 联系客户经理申请开通miniQMT（资金门槛约1-10万）
3. 安装miniQMT客户端
4. 使用xtquant Python库编写交易策略
5. 佣金建议：场内卖出谈到万1，LOF申购费选择打折券商（0.15%）

## 17. 系统运维

### 17.1 启动自检

系统启动时自动检查：
- 数据源连通性（主源+备用源）
- miniQMT客户端连接状态
- SQLite数据库完整性
- 配置文件有效性
- 交易日历是否过期（每年校准一次，或改用上交所官方日历）

### 17.2 心跳与异常告警

- 每5分钟发送心跳日志，确认系统正常运行
- 异常情况立即告警推送：数据源故障、miniQMT断连、策略执行失败等
- 定时任务的二次执行机制：关键任务（如打新申购）若首次失败，5分钟后自动重试一次

## 18. 暂不纳入的策略及原因

| 策略 | 推荐度 | 暂不纳入原因 |
|------|--------|-------------|
| 封闭式基金折价套利 | ⭐⭐⭐ | 资金占用数月到数年，与LOF底仓资金冲突，适合独立仓位参与 |
| REITs打新 | ⭐⭐⭐ | 2025年市场明显降温，已出现破发案例，需精选品种，不能无脑打 |
| ETF套利 | ⭐⭐ | 申赎门槛50-100万份，超出当前资金体量 |
| 期现套利 | ⭐ | 需期货+两融账户，门槛50万+，复杂度极高 |
| 跨市场套利 | - | 高收益机会一旦发现即被限购，验证了"速度即收益"的核心逻辑 |

> 这些策略未来可按需加入。系统设计已预留策略基类和调度器扩展能力，新增策略只需实现策略接口并注册到调度器。
