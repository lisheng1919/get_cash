### 二、资金怎么分配？

假设你的启动资金是 **10万元**（miniQMT开户门槛刚好满足）：

-   **可转债打金**：0元。信用申购，中签后才缴款（一般1000元/签），从LOF底仓里临时调配完全不影响。
-   **节假日套利**：5-10万元。节前一天用闲余资金做逆回购（场内代码204001/131810等），节后第一个交易日资金自动到账，瞬间变回LOF底仓子弹。你的资金只“消失”了一天可用时间，但多赚了过节利息。
-   **LOF底仓套利**：10万元。拿出其中5-8万元建底仓（选2-3只品种分散），留2-5万元作为日常可用资金，刚好用来应付中签缴款或临时加仓LOF。

**资金流大概是这样**：你的10万，平时是LOF底仓；节前瞬间变成逆回购，节后又变回LOF；中间如果中签，划出1000块缴款，20天后卖掉又回到账户。三者完全不在同一时间线冲突。

### 三、技术实现：一个极简集成方案

你不需要为每个策略写一套独立程序。它们完全可以跑在同一个监控系统里：

```
                    ┌─→ 每日9:30 自动打新脚本（一键申购所有待发转债）
                    │
你的AI监控中枢 ────┼─→ 节前/季末 自动提醒+一键逆回购脚本
                    │
                    └─→ 实时溢价监控 → 触发条件 → 弹窗通知你手动卖+申
```

**核心组件清单**：

1.  **可转债打新**：用xtquant写一个极简脚本，每天9:30自动获取当日可申购转债列表，一键提交最大申购额度（100万，反正几乎中不了）。中签结果T+2日自动查询，中签后微信通知你缴款。
2.  **节假日套利**：提前在日历里标记好长假/季末，用脚本计算最佳逆回购品种（通常做1天期204001，但资金占用最少），一键下单。也可以周末自动做场内货基赎回套利（但收益略低）。
3.  **LOF底仓套利**：就是你已经在完善的miniQMT方案，信号触发后手动确认，卖出+申购同步完成。

三者共用同一个miniQMT环境、同一套数据源、同一套通知系统。

### 四、一个被低估的收益增厚细节

你会不会觉得“可转债打新一年才赚两三千，不够塞牙缝”？

别小看这个数字。它的意义不在于绝对值，而在于**风险调整后收益**——你承担的收益波动是零，付出的时间成本是每日3秒，占用的资金成本几乎为零（只在罕见的连续中签时体现）。换算成资金效率，这相当于你凭空多了一笔零成本永续债，年利息2000元。

而节假日套利更有趣：你的10万本金，放在活期里一年利息约300元；如果每次都抓住长假逆回购机会（一年约7次），大概能多赚500-800元，等于活期利息翻倍。这笔钱，本质上是你用“写个定时脚本”的技术投入换来的，边际成本为零。

这两笔钱，加上LOF底仓套利的脉冲收益（一年几次5%溢价机会），你会忽然发现自己的10万本金，年化收益被三个相互独立、互不冲突的微策略，悄悄抬到了一个你自己都意外的水平。

### 五、风险隔离：组合的最大好处不是赚更多，而是“不会同时亏钱”

这三个策略的亏损因素几乎正交：

-   **可转债打新**亏损可能：市场系统暴跌→转债上市破发。但这种环境，通常意味着LOF溢价消失、甚至折价出现，你正好用底仓套利的折价机会（场内买入+赎回）赚钱。
-   **节假日套利**亏损可能：零。国债逆回购场内交易，几乎无风险。
-   **LOF底仓套利**亏损可能：溢价瞬间消失。但你的最大亏损，止于“放弃申购”导致底仓减少（你主动决定不再补回仓位，拿着卖出的现金等待下一次机会），这不是亏损，是你的主动止损。

三者同时亏损的概率，约等于零。这是你组合的最低线——不是一定赚钱，而是几乎不会同时亏钱。
